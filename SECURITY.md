# Security Policy

## Reporting a vulnerability

Please report security vulnerabilities **privately** — do not open a public
issue for an unpatched vulnerability.

Email: **security@ailin.one** — or use [GitHub Private Vulnerability Reporting](https://github.com/ailinone/collective-intelligence/security/advisories/new) for an encrypted, tracked channel

Include: a description of the issue, affected version/commit, reproduction steps
or proof-of-concept, and the impact you believe it has. If you would like an
encrypted channel, mention it in a first (non-sensitive) email and we will
arrange one.

## What to expect

- **Acknowledgement** within a few business days.
- An initial assessment and severity classification.
- Coordinated disclosure: we will agree on a timeline with you and credit you
  (if you wish) once a fix is released.

## Scope

This policy covers the source code in this repository. It does **not** cover
third-party deployments or forks you do not control — if you find an issue in
someone else's hosted instance, report it to that operator.

## Supported versions

Security fixes are applied to the `main` branch and the latest released version.
Older versions are supported on a best-effort basis only.
